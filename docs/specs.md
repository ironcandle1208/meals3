- 機能要件（優先順位：P1…必須機能、P2…できれば実装したい、P3…あれば便利）
    - 献立保存機能（P1）
        - 献立の名前、材料、調理方法を保存（P1）
        - 保存した献立は献立登録画面で一覧として表示される（P1）
    - 献立登録機能（P1）
        - 保存した献立をスケジュールに登録する（P1）
        - 献立は主食・主菜などのタグを付与することができる（P1）
        - 自分でタグを作成できる（P2）
        - 1日は朝食・昼食・夕食で構成される（P1）
    - 買い物リスト機能（P1）
        - 登録した献立に保存されている材料を自動で買い物リストに追加する（P1）
        - 買い物リストの項目は任意の期間を指定すればその期間に登録された献立のものに絞って表示可能（P1）
        - 買い物が完了したら買い物リストの各項目にチェックをつけて未購入/購入済みを管理（P1）
    - スケジュール表示機能（P1）
        - 1日・1週間・1ヶ月単位で献立スケジュールを表示できる（P1）
        - スケジュール画面から1日の献立登録画面に遷移できる（P1）
    - アカウント管理機能（P1）
        - メールアドレスでアカウントを登録できる（P1）
        - 他のアカウントをメールアドレスで検索可能（P1）
    - グループ機能（P1）
        - アカウントを作成するとグループを作成できる（P1）
        - グループには他のアカウントを招待できる（P1）
        - 同一グループに参加しているアカウントと献立・スケジュール・買い物リストを共有できる（P1）
    - AI献立提案機能（P3）
        - AIが献立を提案してくれる（P3）
- 非機能要件
    - モバイルアプリとして開発
        - IOS・Androidの双方でUXが変わらないように設計
    - 金銭的コストは最小限に抑えられるように設計
